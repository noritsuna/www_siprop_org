# B2BUA層
UA間を接続するために必要な動作(実装)を定義する層<br>
SIPとしては、Call(複数のトランザクションやダイアログ)に関して責任を持つ。

## 機能
UAを動作させるための動作手順を記述(プログラミング)する。


# UA層
単一の機能単位で機能(実装)されている層<br>
B2BUAが使用するためのモジュール(API)として、存在している。


## 機能
2つ以上のUAを組み合わせて、1つのUAとして使用するインタプリタ的な動作をする。<br>

```
例：
認証UA⇒アーリートーキUA⇒通話UA
```

というように、単機能を持ったUAを複数使用することにより、通話用UAとして動作させるようなイメージである。<br>
この組み合わせの指定を、B2BUAからプログラミングして、簡易インタプリタで処理している。


# Stack層
SIP Stackとしての機能を持つ層<br>
SIPとしては、トランザクションレベルの処理に対して、責任を持つ。

## 機能
- トランザクションの処理に特化させる
    - それ以上の処理は、UA層に任せる


# Transport層
[MINA](http://mina.apache.org/)を全面的に採用したSocketレベルの処理を扱うための層
